# Icons (placeholder)

E1.2 (Olga, 2026-04-28): the original 1×1 transparent PNGs / zero-byte
`.ico` / `.icns` were rejected by `tauri::generate_context!()` ("icon
32x32.png is not RGBA"), blocking `cargo check --workspace`.

These are now **solid-colour teal RGBA placeholders** generated by a
pure-stdlib Python script — sufficient to make the macro happy and let
the workspace build. They are **not** production artwork.

| File | Size | Format |
|---|---|---|
| `32x32.png` | 32×32 | PNG RGBA |
| `128x128.png` | 128×128 | PNG RGBA |
| `128x128@2x.png` | 256×256 | PNG RGBA |
| `icon.ico` | 32×32 wrapped | ICO (PNG-in-ICO) |
| `icon.icns` | 128×128 wrapped | ICNS (`ic07` entry) |

## Swap-out path (design pass / pre-v1.0)

1. Source a **1024×1024 master `icon.png`** (Catique HUB logo).
2. Run `pnpm tauri icon path/to/icon.png` — Tauri's CLI generates the
   full size set + Windows/macOS containers.
3. Commit the regenerated `icons/`. Delete this README once real icons
   land.

## Regenerate the placeholders (if needed)

```bash
python3 - <<'PY' src-tauri/icons/
# (script lives in /tmp/make_icons.py from E1.2 — copy into repo
```
