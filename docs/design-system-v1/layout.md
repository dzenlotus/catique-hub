## 7. Стратегия брейкпоинтов (Narrow Windows)

Catique HUB — десктопное приложение Tauri. Конкретные narrow-window сценарии:

### Рекомендация: три tier-а ширины

| Tier       | Диапазон     | Поведение                                                          |
|------------|--------------|--------------------------------------------------------------------|
| **Standard** | ≥ 1100 px  | Полный layout: sidebar 220 px + канбан + ≥2 колонки видны         |
| **Compact**  | 900–1099 px | Sidebar сжимается до иконок (48 px wide, без подписей); канбан горизонтально скроллится |
| **Minimum**  | < 900 px   | Tauri запрещает — `minWidth: 900px` в конфиге                    |

**Почему так, а не мобильный стак:**

1. Kanban-доска требует минимум ~560 px для одной колонки (280 px) + sidebar (220 px) + отступы. Ниже 900 px работа невозможна без потери информации.
2. macOS позволяет окна 400 px, но агентский таск-менеджер без sidebar бессмысленен.
3. Horizontal scroll на канбане — стандартное поведение Linear, Notion, Jira — пользователи desktop-приложений ожидают его.

### Compact sidebar (900–1099 px)

```css
@media (max-width: 1099px) {
  .sidebar {
    width: 48px;
    /* Показывать только иконки, скрывать лейблы */
  }
  .sidebar-label,
  .sidebar-wordmark-text {
    display: none;
  }
}
```

- Tooltip при hover на каждой иконке (компонент `Tooltip/`)
- Wordmark сворачивается до логотипа-иконки (32 × 32 px кот)
- Mascot block скрывается (overflow hidden)

### Канбан при узком окне

- `overflow-x: auto` на контейнере — колонки не сжимаются ниже `min-width: 280px`
- Нет горизонтального ресайза колонок в v1
